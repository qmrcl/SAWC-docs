SAWC.doc

Модульная архитектура управления персонажем. Базовая логика Core отделена от вспомогательных модулей (Audio, Visuals, Input). Строгая структура и высокая производительность.

Установка Зависимостей

Для работы контроллера и камеры обязательно требуются следующие пакеты. Без них система выдаст ошибки компиляции.

1. Установка пакетов

Откройте Window -> Package Manager, выберите Unity Registry и установите:

  • Cinemachine (для работы камеры)
  • Input System (для управления)

2. Настройка Input System

Шаг 1: Сразу после установки пакета Unity заорет, что нужен рестарт. Жмите Yes / Restart не думая.

Шаг 2: После перезагрузки топайте в настройки:
Edit -> Project Settings -> Player

Шаг 3: Найдите вкладку Other Settings, раскройте её и скролльте вниз до блока Configuration. Там оно и прячется.

Искомая настройка:
Active Input Handling*
Меняем Input Manager (Old) на:
Both или Input System Package (New)

Советую ставить Both. Так у вас и старые скрипты не отвалятся, и новые будут работать. После смены Unity снова перезагрузится. Это норма.

Рекомендация: Распаковка Префаба (Unpack) Желательно

Для корректной работы UI и системы событий, после размещения префаба игрока на сцене выполните:

1 Найдите игрока в Иерархии
2 Нажмите на него ПКМ
3 Prefab -> Unpack Completely
4 Выделите все вложенные объекты, зажмите ЛКМ и вытащите их в пустое место внизу Иерархии (в корень сцены)
5 Пустой родительский объект теперь не нужен. Выделите его и нажмите Delete, чтобы удалить.
*Это необходимо для предотвращения конфликтов вложенных EventSystem и чтобы не запутаться потому что EventSystem внутри префаба и можно это забыть. Если вы уверены, что никогда не добавите второй EventSystem или будете изменять его только в префабе, то этот шаг можно пропустить — но уже на свой страх и риск.

Расположение компонентов

Все необходимые скрипты и модули находятся в стандартном меню добавления компонентов. Искать их не надо, всё структурировано:

Add Component SAWC Core / Modules

SAWC.Core

Ядро системы. Содержит критически важную логику физики, гравитации и управления состоянием персонажа. Не имеет внешних зависимостей.

SAWC.Modules

Вспомогательные системы: Звук, Камера, Ввод. Модули полностью автономны и могут быть отключены или заменены без нарушения работы ядра.

SAWController

Core Component

Основной контроллер. Вычисляет вектор движения относительно ориентации камеры. Управляет состоянием "на земле" и обрабатывает физику прыжка.

Требования: CharacterController, PlayerInput и камера с тегом "MainCamera".
События (Events)
Событие Описание Применение
JumpPerformed
 Событие отрыва от поверхности (прыжок). Воспроизведение звука прыжка, эффекты частиц.
LandPerformed
 Событие приземления (IsGrounded = true). Звук приземления, эффект тряски камеры, анимация посадки.
StartMoving
 Начало движения (Скорость > 0). Запуск анимации ходьбы, воспроизведение шагов.
StopMoving
 Остановка движения. Переход в Idle, остановка звуков.
Публичный API
Имя Описание
IsMoving Состояние движения персонажа.
IsGrounded Находится ли персонаж на земле (CharacterController).
Jump() Выполнить принудительный прыжок (для скриптовых сцен).

CharacterSettings

ScriptableObject

Movement (Движение)

CanJump / CanSprint bool

Глобальные переключатели. Отключают возможность прыгать или бегать, независимо от ввода игрока. Полезно для дебаффов или сюжетных ограничений.

EnableAutoJump bool

Если включено, персонаж будет прыгать без остановки, пока зажата клавиша прыжка (Bunny Hop). Если выключено — нужно нажимать каждый раз.

MoveSpeed / SprintSpeed float

Целевая скорость перемещения. SprintSpeed не может быть меньше MoveSpeed (валидация в OnValidate).

JumpForce float

Начальная вертикальная скорость при прыжке. Чем больше, тем выше летит.

Physics & Air Control

Gravity float

Сила притяжения в полете. Обычно -9.81. Всегда отрицательное значение.

TerminalVelocity float

Максимальная скорость падения (например, -50). Не дает персонажу провалиться сквозь пол при долгом падении.

Acceleration float

Скорость набора скорости. Высокое значение = отзывчивое управление. Низкое = эффект инерции/тяжести.

Deceleration float

Скорость торможения при отпускании кнопок. Если меньше Acceleration, персонаж будет "скользить" при остановке.

AirControlMultiplier float (0-1)

Насколько сильно можно менять направление полета в воздухе. 0 = летишь по инерции как камень. 1 = управляешь как на земле.

Rotation (Вращение)

RotateWithMovement bool

Главный переключатель режима камеры.
True: Персонаж смотрит туда, куда бежит (Adventure).
False: Персонаж всегда смотрит вперед от камеры (Strafe).

RotationSmoothTime float

Время сглаживания поворота (Damp). Чем больше значение, тем медленнее и плавнее поворачивается моделька.

Advanced Physics

GroundedGravity float

Гравитация, применяемая когда персонаж стоит на земле (например, -2). Нужна, чтобы CharacterController "прилипал" к наклонным поверхностям и ступенькам.

FallMultiplier float

Множитель гравитации при падении (Velocity < 0). Делает падение быстрее взлета, что ощущается "тяжелее" и реалистичнее в играх.

Device Detector

Input System

Стратегии Определения Устройства

Используется паттерн Strategy. Детектор последовательно проверяет список объектов DeviceDetectionStrategy, пока не будет определен тип устройства.

Важность порядка

Детектор проверяет стратегии сверху вниз. Располагайте более специфичные проверки (например, Gamepad) выше общих проверок (например, Platform Check).

Создание собственной стратегии

1. Создайте скрипт, наследуемый от DeviceDetectionStrategy.
2. Переопределите метод Detect(), возвращая тип устройства.
3. Создайте ассет через меню Create -> SAWC.
4. Добавьте ассет в список детектора на сцене.

Touch Controls

InvertedLookPad

Умный Тачпад

Компонент управления камерой на базе EnhancedTouch API.

Отслеживание ID

Система запоминает палец управления камерой, игнорируя касания джойстика перемещения.

Мертвая зона

Позволяет исключить область (RectTransform) из обработки касаний камеры (например, область стика).

Адаптация DPI

Чувствительность автоматически масштабируется в зависимости от плотности пикселей экрана.

Audio & Camera

PlayerAudioController

Автоматический контроллер звука. Подписывается на события движения.

  • Step (Loop) — воспроизводится при движении по земле.
  • Jump / Land (OneShot) — воспроизводятся однократно.

CameraShakeControl

Интеграция с Cinemachine. Управляет параметрами шума камеры.

При ускорении (спринт), параметры шума плавно интерполируются (SmoothDamp) к более высоким значениям.

Увеличение скорости = усиление эффекта тряски.
